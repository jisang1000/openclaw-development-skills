# OpenClaw Development Skills

A practical skill set for disciplined development work in OpenClaw.

This collection adapts proven agent-development patterns into OpenClaw-friendly skills focused on:
- planning before execution
- systematic debugging
- verifying behavior before declaring completion
- reviewing quality before handoff
- using test-first thinking when behavior should be proven
- managing branch hygiene and parallel execution safely

## Core Skills

- `verification-before-completion`
- `systematic-debugging`
- `writing-plans`
- `requesting-code-review`
- `test-driven-development`
- `development-workflow`

## Supporting Skills

- `using-git-worktrees`
- `finishing-a-development-branch`
- `dispatching-parallel-agents`
- `executing-plans`

## Recommended Adoption Order

1. verification-before-completion
2. systematic-debugging
3. writing-plans
4. requesting-code-review
5. test-driven-development
6. development-workflow
7. supporting workflow skills

## What This Set Is Good For

- tool and runtime setup
- automation repair
- browser/script debugging
- structured implementation work
- safer multi-step development tasks
- more reliable final reporting

## Design Principle

These skills are intentionally lightweight and behavior-oriented.
They do not try to replace OpenClaw's tools. Instead, they improve how an agent uses those tools.

## First Publish Recommendation

Start by publishing `verification-before-completion` first.
It is the easiest to explain, broadly useful, and representative of the set's value.
